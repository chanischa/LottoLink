### 1. ความสอดคล้อง (Alignment) — 30 คะแนน

> นวัตกรรมของคุณช่วยแก้ปัญหา (Pain Point) อะไรให้ GLO และสอดคล้องกับโจทย์ที่คุณเลือกในส่วนที่ 2 อย่างไร?

LottoLink ถูกออกแบบมาเพื่อแก้ปัญหาหลักที่ GLO เผชิญอยู่โดยตรง 3 ระดับ ซึ่งสอดคล้องกับโจทย์ทั้ง 3 ธีมของการแข่งขัน:

**Pain Point ที่ 1 — ภาระงาน Manual สูงเกินไป (ตรงกับ Smart Work Automation)**
GLO รับมือกับผู้มา Walk-in รับรางวัลมากกว่า 20,000 รายต่อเดือน โดยเจ้าหน้าที่ต้องตรวจสอบและดำเนินการด้วยมือทุกขั้นตอน LottoLink แก้ปัญหานี้โดยให้ระบบตรวจสอบความถูกต้องของลอตเตอรี่, ลงทะเบียนกรรมสิทธิ์, ตรวจผลรางวัล และโอนเงินรางวัลแบบอัตโนมัติ โดยไม่ต้องใช้เจ้าหน้าที่สำหรับรายที่ความเสี่ยงต่ำ

**Pain Point ที่ 2 — กระบวนการซ้ำซ้อนและขาด Traceability (ตรงกับ Process Automation)**
กระบวนการขอรับรางวัลปัจจุบันเต็มไปด้วยขั้นตอนซ้ำซ้อน ไม่มีระบบติดตามกรรมสิทธิ์ และไม่มี Audit Trail ที่เชื่อถือได้ LottoLink เปลี่ยนทุก Workflow เหล่านี้ให้เป็นดิจิทัล ตรวจสอบได้ และบันทึกอัตโนมัติ

**Pain Point ที่ 3 — กระบวนการพึ่งพากระดาษและการเดินทาง (ตรงกับ Sustainable Eco-Automation)**
การที่ประชาชนต้องเดินทางมา Walk-in และกรอกแบบฟอร์มกระดาษทุกครั้งสร้างทั้ง Carbon Footprint และสิ้นเปลืองทรัพยากร LottoLink ให้รับรางวัลจากที่บ้านผ่านมือถือ ยกเลิกแบบฟอร์มกระดาษ และโอนเงินผ่านระบบ Cashless อัตโนมัติ

**ความสอดคล้องโดยสรุป:**

| โจทย์การแข่งขัน | Pain Point ของ GLO | วิธีที่ LottoLink แก้ |
|---|---|---|
| Smart Work Automation | 20,000+ Walk-in / เดือน, 17 นาที/ราย | ตรวจสอบและโอนรางวัลอัตโนมัติ |
| Process Automation | ซ้ำซ้อน, ไม่มี Audit Trail, ข้อพิพาทกรรมสิทธิ์ | Digital Ownership Lock + Full Audit Log |
| Sustainable Eco-Automation | กระดาษ, การเดินทาง, Cash-only | Cashless Payout + Self-service ทุกที่ |

### 2. ความสร้างสรรค์ (Innovation) — 30 คะแนน

> ผลงานของคุณมีความใหม่ แตกต่าง หรือล้ำหน้ากว่าระบบ/กระบวนการเดิมที่ GLO ใช้อยู่ในปัจจุบันอย่างไร?

**สิ่งที่ไม่เคยมีในระบบ GLO ปัจจุบัน:**

**1. Digital Ownership Lock — กรรมสิทธิ์ดิจิทัลที่ยืนยันไม่ได้**
ปัจจุบัน GLO ไม่มีระบบลงทะเบียนกรรมสิทธิ์ลอตเตอรี่ก่อนถึงวันรับรางวัล LottoLink สร้างหลักการ **"One Ticket = One Verified Owner = One Reward Claim"** ที่ล็อกกรรมสิทธิ์ทันทีที่สแกน ทำให้ไม่มีใครแย่งสิทธิ์หรืออ้างสิทธิ์ซ้ำได้อีก — นี่คือนวัตกรรมที่ไม่เคยมีในระบบสลากไทย

**2. AI Fraud Scoring แบบ Real-time ก่อนการจ่ายเงิน**
ระบบปัจจุบันตรวจสอบการทุจริตหลังจากที่มีการร้องเรียน (Reactive) LottoLink ใช้ AI วิเคราะห์ความเสี่ยงทุกรายการก่อนโอนเงิน (Proactive) โดยประเมินจากรูปแบบการสแกน, ความผิดปกติของอุปกรณ์, และ Pattern การปลอมแปลงภาพ — เปลี่ยนจาก "รอเกิดปัญหา" เป็น "ป้องกันก่อนเกิด"

**3. End-to-End Automation บน Platform ที่มีผู้ใช้แล้ว 40+ ล้านราย**
LottoLink ไม่ได้สร้าง App ใหม่ที่ต้องให้ประชาชน Download ใหม่ แต่ฝังระบบทั้งหมดใน **เป๋าตัง** ซึ่งเชื่อมต่อกับ National Digital ID, PromptPay, และระบบธนาคารแล้ว ทำให้ Implementation เร็วกว่าและ Adoption สูงกว่าการสร้าง Platform ใหม่ตั้งแต่ศูนย์

**4. Automated Payout ข้ามช่องทางอัตโนมัติ**
ไม่มีระบบใดของ GLO ในปัจจุบันที่สามารถโอนเงินรางวัลให้ผู้ถูกรางวัลโดยตรงโดยไม่ต้องมาที่สำนักงาน LottoLink เป็นระบบแรกที่ทำให้กระบวนการนี้เป็น Zero-touch สำหรับรายที่ผ่าน Fraud Check

**ตารางเปรียบเทียบความล้ำหน้า:**

| มิติ | GLO ปัจจุบัน | LottoLink |
|---|---|---|
| การตรวจสอบลอตเตอรี่ | Manual โดยเจ้าหน้าที่ | Automated ภายใน 3 วินาที |
| กรรมสิทธิ์ | ไม่มีระบบก่อนวันรับรางวัล | Digital Lock ทันทีที่สแกน |
| การตรวจจับทุจริต | Reactive (หลังร้องเรียน) | Proactive AI (ก่อนโอนเงิน) |
| การรับรางวัล | Walk-in เท่านั้น | Remote Automated Payout |
| Audit Trail | กระดาษ / Manual | Digital Log ทุก Event อัตโนมัติ |
| Platform | Standalone System | Embedded in เป๋าตัง (40M+ users) |

### 3. ความเป็นไปได้ (Feasibility) — 20 คะแนน

> นวัตกรรมนี้สามารถนำมาเชื่อมต่อหรือใช้งานจริงกับโครงสร้างพื้นฐาน/บุคลากรของ GLO ได้จริงหรือไม่ อย่างไร?

**ใช้งานได้จริง — เพราะ LottoLink ต่อยอดจากสิ่งที่มีอยู่แล้ว ไม่ใช่สร้างใหม่ทั้งหมด**

**โครงสร้างพื้นฐานที่พร้อมรองรับ:**

| ระบบที่มีอยู่แล้ว | บทบาทใน LottoLink |
|---|---|
| **เป๋าตัง (Paotang)** | UI หลักสำหรับผู้ใช้ — ไม่ต้องพัฒนา App ใหม่ |
| **National Digital ID (NDID)** | ยืนยันตัวตนเจ้าของลอตเตอรี่ — มีอยู่ใน เป๋าตัง แล้ว |
| **PromptPay ของธนาคารกรุงไทย** | ช่องทางโอนเงินรางวัลอัตโนมัติ — เชื่อมต่อแล้ว |
| **ฐานข้อมูลลอตเตอรี่ GLO** | ให้ข้อมูล Serial / Draw Results ผ่าน API |
| **QR Code บนลอตเตอรี่** | ใช้สำหรับสแกนและตรวจสอบ — มีอยู่บนลอตเตอรี่ปัจจุบันแล้ว |

**สิ่งที่ GLO ต้องทำเพิ่มเพียง 2 อย่าง:**
1. เปิด API สำหรับ Serial Validation และ Draw Result Feed ให้ระบบ LottoLink เรียกใช้
2. กำหนด Business Rule สำหรับ Payout Threshold และ Manual Review Process

**ผลกระทบต่อบุคลากร GLO:**
- เจ้าหน้าที่ไม่ถูกแทนที่ แต่เปลี่ยนบทบาทจาก "ผู้ตรวจสอบทุกราย" เป็น "ผู้จัดการเคสความเสี่ยงสูง"
- ระบบ Claim Review Workflow ช่วยให้เจ้าหน้าที่มี Queue ที่มีคุณภาพ — เฉพาะรายที่จำเป็นต้องตรวจ
- GLO Dashboard ให้ภาพรวม Real-time ที่ทีม Operation สามารถติดตามได้ทันที

**ความเสี่ยงที่บริหารจัดการได้:**

| ความเสี่ยง | วิธีรับมือ |
|---|---|
| API Integration กับ GLO ล่าช้า | เริ่มด้วย Mock API ใน Prototype, ประสาน GLO ในระยะ Pilot |
| ผู้ใช้ไม่คุ้นเคยกับระบบดิจิทัล | UX ใน เป๋าตัง ออกแบบเรียบง่าย, มีช่องทาง Walk-in สำรองยังคงอยู่ |
| ความปลอดภัยของข้อมูล | ใช้โครงสร้างความปลอดภัยของ เป๋าตัง / KTB ที่ผ่านมาตรฐานธนาคารแห่งประเทศไทยอยู่แล้ว |

### 4. ผลกระทบ (Impact) — 20 คะแนน

> ระบุผลลัพธ์เชิงบวกที่เป็นรูปธรรม (เช่น ตัวเลขการลดเวลา, ลดค่าใช้จ่าย, การเพิ่มประสิทธิภาพ หรือประโยชน์ต่อสังคม/สิ่งแวดล้อม)

**ผลลัพธ์เชิงตัวเลข (Quantified Impact):**

| ตัวชี้วัด | ก่อน LottoLink | หลัง LottoLink (เป้าหมาย) |
|---|---|---|
| Walk-in รับรางวัลต่อเดือน | 20,000+ ราย | ลดเหลือ 4,000–6,000 ราย (↓ 70–80%) |
| เวลาดำเนินการต่อราย | 17 นาที | ต่ำกว่า 2 นาที สำหรับ Auto-route (↓ 88%) |
| รายการที่ต้องใช้เจ้าหน้าที่ | 100% ของทุกรายการ | เฉพาะ ~25% (เคสความเสี่ยงปานกลาง-สูง) |
| แบบฟอร์มกระดาษต่อเดือน | 20,000+ ชุด | 0 สำหรับ Digital Claims |
| เวลารอรับเงินรางวัล (ต่ำกว่า 20,000 บาท) | 1–3 วันทำการ | ภายใน 15 นาที (Auto PromptPay) |
| การตรวจจับทุจริต | Reactive (หลังร้องเรียน) | Proactive Real-time ก่อนจ่ายทุกรายการ |

**ผลกระทบด้านการเงินของ GLO:**
- ค่าใช้จ่ายในการจัดการ Walk-in ลดลงตามสัดส่วนของปริมาณงาน
- ลดความเสียหายจากการจ่ายรางวัลให้ลอตเตอรี่ปลอม ซึ่งยากต่อการประเมินมูลค่าแต่ Fraud Detection แบบ Proactive ป้องกันได้ก่อนเกิดความเสียหาย

**ผลกระทบต่อประชาชน:**
- ประหยัดเวลา 1–2 ชั่วโมงต่อคน (การเดินทาง + รอคิว) สำหรับผู้ถูกรางวัล 14,000–16,000 รายต่อเดือนที่ใช้ระบบดิจิทัล
- ผู้สูงอายุและผู้อยู่ต่างจังหวัดเข้าถึงการรับรางวัลได้โดยไม่ต้องเดินทาง — ลดความเหลื่อมล้ำในการเข้าถึงบริการของรัฐ
- ลดความเสี่ยงจากลอตเตอรี่ปลอมสำหรับผู้ซื้อ เพราะสามารถยืนยันความถูกต้องก่อนซื้อหรือทันทีหลังซื้อ

**ผลกระทบด้านสิ่งแวดล้อม:**
- ลดการใช้กระดาษจากแบบฟอร์มและเอกสารคิวประมาณ **20,000+ ชุดต่อเดือน**
- ลด Carbon Emission จากการเดินทางของผู้รับรางวัล 14,000–16,000 เที่ยวต่อเดือน (โดยเฉลี่ยประมาณ 20–30 กม./เที่ยว)
- ลดความต้องการพื้นที่ให้บริการ Walk-in และอุปกรณ์สำนักงานตามปริมาณงานที่ลดลง

**ผลกระทบระยะยาว:**
LottoLink สร้างโครงสร้างพื้นฐาน Digital Ownership Verification ที่สามารถขยายต่อยอดสู่ระบบตั๋วอีเวนต์, คูปองรัฐบาล, และ Digital Voucher Systems — เพิ่มคุณค่าให้กับ GLO และรัฐบาลไทยในระยะยาวเกินกว่าโครงการสลากเพียงอย่างเดียว

## 1. สรุปแนวคิดโครงการ (Executive Summary)

**LottoLink** คือแพลตฟอร์มดิจิทัลสำหรับการตรวจสอบลอตเตอรี่และรับรางวัลอัตโนมัติ พัฒนาบนแอปพลิเคชัน **เป๋าตัง** ของธนาคารกรุงไทย

**ปัญหาในภาพรวม:**
ปัจจุบันสำนักงานสลากกินแบ่งรัฐบาลต้องรับมือกับผู้ขอรับรางวัลแบบ Walk-in มากกว่า 20,000 รายต่อเดือน ใช้เวลาเฉลี่ย 17 นาทีต่อคิว กระบวนการทั้งหมดยังเป็นแบบ Manual พร้อมความเสี่ยงจากการปลอมแปลงและการอ้างสิทธิ์ซ้ำ

**วิธีการแก้ปัญหา:**
LottoLink ให้ผู้ใช้สแกนลอตเตอรี่ผ่านแอปเป๋าตัง ระบบจะตรวจสอบความถูกต้อง ลงทะเบียนกรรมสิทธิ์ดิจิทัล ตรวจผลรางวัลอัตโนมัติ และโอนเงินรางวัลผ่าน PromptPay / บัญชีธนาคาร โดยไม่ต้องเดินทางมาที่สำนักงาน

**ผลลัพธ์ที่คาดว่าจะได้รับ:**
- ลด Walk-in ที่สำนักงานสลากฯ ได้ 70–80%
- ลดเวลาดำเนินการต่อรายจาก 17 นาที เหลือต่ำกว่า 2 นาที
- ป้องกันการทุจริตและลอตเตอรี่ปลอมด้วย AI แบบ Real-time
- ลดการใช้กระดาษและทรัพยากรในกระบวนการขอรับรางวัล

## 2. หลักการและเหตุผล (Problem Statement)

**ปัญหาที่ต้องการแก้ไข:**

กระบวนการขอรับรางวัลลอตเตอรี่ของไทยยังคงเป็นระบบ Manual ที่ต้องอาศัยการเดินทางมาที่สำนักงานสลากกินแบ่งรัฐบาล ซึ่งก่อให้เกิดปัญหาหลัก ดังนี้

| ปัญหา | ผลกระทบ |
|---|---|
| ผู้ขอรับรางวัลต้องมา Walk-in มากกว่า 20,000 รายต่อเดือน | คิวยาว เสียเวลา เพิ่มภาระพนักงาน |
| กระบวนการตรวจสอบด้วยมือใช้เวลาเฉลี่ย 17 นาทีต่อราย | ขาดประสิทธิภาพ ไม่สามารถรองรับปริมาณงานที่เพิ่มขึ้น |
| ไม่มีระบบตรวจสอบความเป็นเจ้าของ | เกิดข้อพิพาท การอ้างสิทธิ์ซ้ำ และการทุจริตได้ง่าย |
| ลอตเตอรี่ปลอมตรวจจับยาก | สร้างความเสียหายทางการเงินและความน่าเชื่อถือของระบบ |
| ไม่มีระบบแจ้งผลรางวัลอัตโนมัติ | ผู้ถูกรางวัลอาจพลาดการรับรางวัลภายในกำหนด |

**กลุ่มเป้าหมาย:**
1. **ผู้ซื้อลอตเตอรี่** — ประชาชนทั่วไปที่ต้องการรับรางวัลอย่างสะดวกและปลอดภัย
2. **สำนักงานสลากกินแบ่งรัฐบาล (GLO)** — ต้องการลดภาระงาน Manual และเพิ่มประสิทธิภาพ
3. **ผู้ขายลอตเตอรี่** — ต้องการเครื่องมือยืนยันความถูกต้องของสลากก่อนขาย

**ผลกระทบของปัญหา:**
- ด้านเศรษฐกิจ: สูญเสียทรัพยากรบุคคลและงบประมาณในการบริหารจัดการคิวและเอกสาร
- ด้านสังคม: ประชาชนเสียเวลาและค่าใช้จ่ายในการเดินทาง โดยเฉพาะผู้สูงอายุและผู้อยู่ต่างจังหวัด
- ด้านความโปร่งใส: ขาดระบบตรวจสอบย้อนหลังที่เชื่อถือได้ ทำให้การทุจริตเกิดขึ้นได้ยาก

## 3. รายละเอียดโซลูชันและนวัตกรรม (Proposed Solution & Innovation)

**ผลงานคืออะไร:**
LottoLink คือระบบดิจิทัลที่แปลงลอตเตอรี่กระดาษให้กลายเป็น Digital Asset ที่มีการยืนยันกรรมสิทธิ์และประมวลผลรางวัลอัตโนมัติ ผ่านแอปเป๋าตังที่ประชาชนไทยใช้งานอยู่แล้ว

**การทำงานของระบบ:**

```
ผู้ใช้สแกน QR / Serial บนลอตเตอรี่
            ↓
ระบบตรวจสอบความถูกต้อง (Authenticity Check)
            ↓
ลงทะเบียนกรรมสิทธิ์ดิจิทัล (Ownership Lock)
            ↓
เก็บลอตเตอรี่ใน Digital Wallet ภายในเป๋าตัง
            ↓
ระบบตรวจผลรางวัลอัตโนมัติเมื่อประกาศผล
            ↓
AI วิเคราะห์ความเสี่ยงการทุจริต
            ↓
[ความเสี่ยงต่ำ] → โอนรางวัลอัตโนมัติ PromptPay / ธนาคาร
[ความเสี่ยงปานกลาง–สูง] → ส่งให้เจ้าหน้าที่ตรวจสอบ
```

**เทคโนโลยี Smart Automation ที่ใช้:**

| เทคโนโลยี | การนำไปใช้ใน LottoLink |
|---|---|
| **AI / Machine Learning** | วิเคราะห์ภาพลอตเตอรี่เพื่อตรวจจับการปลอมแปลง, คำนวณ Fraud Risk Score แบบ Real-time |
| **Digital Verification Automation** | ตรวจสอบ QR Code และ Serial Number อัตโนมัติ เปรียบเทียบกับฐานข้อมูล GLO |
| **Process Automation (RPA-style)** | ตรวจผลรางวัล ประมวลผลสิทธิ์ และโอนเงินรางวัลโดยไม่ต้องใช้คนดำเนินการ |
| **Financial API Integration** | เชื่อมต่อ PromptPay API, Bank API และ Paotang Wallet สำหรับโอนเงินอัตโนมัติ |
| **Audit Trail Automation** | บันทึกทุก Event อัตโนมัติ — กรรมสิทธิ์, การสแกน, การรับรางวัล — เพื่อความโปร่งใส |

**ความใหม่และความแตกต่างจากที่มีอยู่:**

| มิติ | ระบบปัจจุบัน | LottoLink |
|---|---|---|
| การตรวจสอบลอตเตอรี่ | Manual โดยเจ้าหน้าที่ | อัตโนมัติผ่านแอป ภายในไม่กี่วินาที |
| การลงทะเบียนกรรมสิทธิ์ | ไม่มี | Digital Ownership Lock — One Ticket = One Owner |
| การตรวจผลรางวัล | ผู้ใช้ตรวจเอง | ระบบตรวจและแจ้งเตือนอัตโนมัติ |
| การรับรางวัล | Walk-in เท่านั้น | โอนอัตโนมัติผ่าน PromptPay / ธนาคาร |
| การป้องกันทุจริต | ไม่มีระบบ Proactive | AI Fraud Scoring ก่อนโอนเงินทุกครั้ง |
| Audit Trail | กระดาษ / Manual log | Digital log ครบทุก Transaction อัตโนมัติ |

## 4. แผนการพัฒนาต้นแบบและระยะเวลา (Implementation Plan for Prototype)

**เป้าหมายของ Prototype:**
สาธิตการทำงานครบ End-to-End ตั้งแต่การสแกนลอตเตอรี่จนถึงการรับรางวัลอัตโนมัติ บน Mock Environment ที่จำลองระบบ GLO และ PromptPay

**แผนงาน 8 สัปดาห์:**

| สัปดาห์ | งาน | ผลลัพธ์ |
|---|---|---|
| **สัปดาห์ที่ 1–2** | ออกแบบ Database Schema, API Contract, และ UI Wireframe | Schema, API Spec, Design Mockup |
| **สัปดาห์ที่ 3–4** | พัฒนา Ticket Verification Module (QR Scan + Serial Validation) | สแกนและตรวจสอบลอตเตอรี่ได้จริง |
| **สัปดาห์ที่ 5** | พัฒนา Ownership Management + Digital Wallet | ลงทะเบียนกรรมสิทธิ์และเก็บลอตเตอรี่ใน Wallet |
| **สัปดาห์ที่ 6** | พัฒนา Auto Reward Processing + Fraud Risk Scoring (Rule-based) | ระบบตรวจผลและประมวลผลรางวัลอัตโนมัติ |
| **สัปดาห์ที่ 7** | Integration Testing + Mock Payout Flow + Operational Dashboard | Prototype ครบ Flow ทดสอบได้จริง |
| **สัปดาห์ที่ 8** | QA, Bug Fix, Demo Preparation, Pitch Deck | Prototype พร้อม Demo และนำเสนอ |

**Prototype Scope (MVP):**
- สแกน QR / Serial ของลอตเตอรี่จำลอง
- ลงทะเบียนกรรมสิทธิ์และ Lock อัตโนมัติ
- ตรวจผลรางวัลจาก Mock GLO Feed
- แสดง Fraud Risk Score และ Routing Decision
- จำลองการโอนรางวัลอัตโนมัติ (Mock PromptPay)
- Dashboard แสดง Metrics สำหรับ GLO

**ทีมงานและโมดูลรับผิดชอบ:**

| โมดูล | รายละเอียด |
|---|---|
| Ticket Verification | QR Scan, Serial Validation, Authenticity Check |
| Ownership & Wallet | Digital Registration, Lock, Wallet Storage |
| Reward & Fraud | Auto Result Check, Risk Scoring, Payout Routing |
| Dashboard & Integration | GLO Dashboard, Mock API Integration, Demo Setup |

## 5. ผลกระทบและคุณค่าต่อสังคม/ผู้ใช้ (Social Impact & Value)

**ผลกระทบเชิงบวกตามแนวคิด People & Planet:**

### People — คุณค่าต่อประชาชนและสังคม

| มิติ | ผลกระทบ |
|---|---|
| **ลดเวลา** | ผู้รับรางวัลไม่ต้องเสียเวลา 17 นาทีต่อคิว และไม่ต้องเดินทางมาที่สำนักงาน ประหยัดเวลาและค่าใช้จ่ายในการเดินทาง |
| **เพิ่มความสะดวก** | รับรางวัลได้จากทุกที่ผ่านสมาร์ทโฟน ไม่จำกัดวันเวลาทำการ |
| **ความเท่าเทียม** | ผู้สูงอายุ ผู้พิการ และผู้อยู่ต่างจังหวัดเข้าถึงการรับรางวัลได้โดยไม่ต้องเดินทาง |
| **ป้องกันการทุจริต** | ระบบ One Ticket = One Owner ปิดช่องโหว่การอ้างสิทธิ์ซ้ำและลอตเตอรี่ปลอมได้อย่างเป็นระบบ |
| **เพิ่มความโปร่งใส** | ทุก Transaction บันทึก Audit Log อัตโนมัติ ตรวจสอบได้ตลอดเวลา เพิ่มความเชื่อมั่นของสาธารณชนต่อ GLO |
| **Financial Inclusion** | รับรางวัลผ่าน PromptPay และบัญชีธนาคาร ลดการพึ่งพาเงินสด ขยายการเข้าถึงทางการเงิน |

### Planet — คุณค่าต่อสิ่งแวดล้อม

| มิติ | ผลกระทบ |
|---|---|
| **ลดการใช้กระดาษ** | ยกเลิกแบบฟอร์มขอรับรางวัลกระดาษ ใบเสร็จกระดาษ และเอกสารคิวสำหรับ 20,000+ รายต่อเดือน |
| **ลด Carbon Footprint** | การลด Walk-in ลง 70–80% ตัดการเดินทางของผู้รับรางวัลออก ลด Emission จากการขนส่ง |
| **ลดการใช้พลังงานของสำนักงาน** | ปริมาณงานที่ลดลงหมายถึงพื้นที่สำนักงาน แสงสว่าง และระบบปรับอากาศที่ต้องการน้อยลง |
| **Digital-First Infrastructure** | แทนที่การเก็บเอกสารกระดาษด้วย Digital Audit Log ลดความต้องการพื้นที่จัดเก็บและวัสดุสิ้นเปลือง |
